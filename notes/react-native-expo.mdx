### Overview

[React Native](https://reactnative.dev): UI framework for writing native Android, iOS, UWP, Web, etc apps using JavaScript and React.
- Since 2024, RN has had the ["New Arhictecture"](https://reactnative.dev/architecture/landing-page) available (which I think is enabled by default now on newer RN version) that supports concurrent rendering, automatic render batching, replaces the asynchronous bridge b/w JS and native with JSI allowing direct invocation of native methods without serialization cost.

[Expo](https://expo.dev): suite of open source SDKs and libraries for native functionality (GPS, bluetooth, camera, etc), file-based routing (they wrote React Navigation), and dev CLI/tooling to tie it all together.
- See the big ol' list of native libraries provided by the [Expo SDK](https://docs.expo.dev/versions/latest/sdk/expo/).

### Dev tooling
- [React Native Dev Tools](https://reactnative.dev/docs/react-native-devtools) – these are bundled and require chrome to be installed. Launched by pressing “j” in the terminal that’s running the react native app.
- [Expo Application Services (EAS)](https://expo.dev/eas): tooling for running cloud builds, submitting to app stores, OTA javascript updates, GH actions integration, etc.
  - OTA updates only applicable for updates to the JS layer (ie no native code changes).
- [Appetize.io](https://appetize.io): service for running android/iOS emulators from the browser - haven’t experimented with it yet, but has an EAS integration.

### Misc dev notes / other stuff to look at
- [react-native-vision-camera](https://github.com/mrousavy/react-native-vision-camera): full featured camera library with all the bells and whistles!
- [react-native-gifted-chat](https://github.com/FaridSafi/react-native-gifted-chat?tab=readme-ov-file): complete chat UI! this has been around for like 8+ years, so it's cool to see it still being supported and worked on!
- Platform specific components/modules: 
  - You can write platform-specific components by adding separate files with the platform extension, e.g. “*.android.tsx, *.ios.tsx, *.web.tsx”
    - https://docs.expo.dev/router/advanced/platform-specific-modules/ 
  - Or you can import the [Platform module from React Native](https://reactnative.dev/docs/platform-specific-code) to detect the current platform and conditionally render based on that
    - e.g. `if (Platform.OS === ‘android”) { // do android stuff }`
- [Writing native modules using the Expo Modules API](https://docs.expo.dev/workflow/customizing/#writing-native-code)
- Expo Router has protected routes, which [provide a “guard” property](https://expo.dev/blog/simplifying-auth-flows-with-protected-routes)
- [yoga](https://github.com/facebook/yoga) is a flexbox layout engine, and i'm pretty sure is core to how styles are defined in RN and why stylesheet uses a subset of flex
- EAS has a [push notifications service](https://docs.expo.dev/push-notifications/overview/), so unified API for communicating to both platforms (Firebase Cloud Messaging vs Apple Push Notification Service)
